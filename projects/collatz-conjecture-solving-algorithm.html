About This Project

This project aims to demonstrate the Collatz Conjecture, a mathematical hypothesis that suggests that for any positive integer, repeatedly applying a simple set of rules will always result in the number 1.

“ The conjecture asks whether repeating two simple arithmetic operations will eventually transform every positive integer into 1. ”
Wikipedia

The conjecture is named after the German mathematician Lothar Collatz, who first proposed it in 1937. The basic rule of the conjecture is as follows: if the number is even, divide it by 2; if it is odd, multiply it by 3 and add 1. This process is repeated for each new number generated.

110 is even → 10÷2 = 5
25 is odd → 5*3+1 = 16
316 is even → 16÷2 = 8
48 is even → 8÷2 = 4
54 is even → 4÷2 = 2
62 is even → 2÷2 = 1

As we can see, when we apply these rules to the number 10, we will get 1. Same goes for all numbers smaller than 2.95*1020, which is the largest number that has been tested yet. Despite its simplicity, the conjecture has not been proven for all integers, making it one of the unsolved problems in mathematics.

This project is designed to verify the Collatz Conjecture for a large range of integers. The algorithm will repeatedly apply the rules of the conjecture to each number in a given range to determine whether it eventually reaches 1 or not.

To use this program, clone the nymvaclav/collatz_conjecture_solving_algorithm repository to your device [see: how to clone a Git repository], run the setup.py file to configure your preferences, and then run the main.py file to start the algorithm and begin testing integers.

How to Use This Application

The application is written in Python and is not pre-compiled, so you will need to have Python 3 or a newer version installed on your system. To get started, first clone the repository to your local machine. Then, follow the setup instructions you will see in the terminal after running the setup.py file, where you can specify key parameters, such as the starting integer, the range of integers to test, how many iterations to perform, and which integers to exclude from the calculations, etc. Here is a sample configuration of the preferences.json file:

1{
2 "start_from_int": 236763427863786283,
3 "how_many_int_to_check": 100,
4 "stop_at_int": 0,
5 "stop_if_loop_is_found": 2,
6 "exclude_int": [],
7 "create_logs": true,
8 "print_info": false,
9 "print_operations": false,
10 "print_results": 3
11}

Once you've configured the preferences, simply run the main.py file to start the calculations. The program will apply the Collatz operations to the numbers in the range you've specified, providing outputs based on your preferences.
